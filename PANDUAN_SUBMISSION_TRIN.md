## 1. Struktur Kriteria Hierarki

Sistem akreditasi IABEE 2026 menggunakan **hierarki 3-level** kriteria:

### Level 0: Kriteria Utama (4 item)

| Kode | Nama | Deskripsi | Bobot |
|------|------|-----------|-------|
| **K1** | Orientasi Kompetensi Lulusan | Penetapan, penyebaran, kaji ulang PPM & CPL | 28 |
| **K2** | Implementasi Pembelajaran | Kurikulum, dosen, mahasiswa, fasilitas | 45 |
| **K3** | Asesmen Capaian Pembelajaran | Proses pengukuran CPL & jaminan kelulusan | 14 |
| **K4** | Perbaikan Berkelanjutan | Evaluasi berkala & tindak lanjut mutu | 13 |
| | **TOTAL** | | **100** |

### Level 1: Sub-Grup (Pengelompok, 10 item)

**K1 — 3 sub-grup:**
- K1.1: Penetapan PPM (bobot 2)
- K1.2: Penyebaran PPM & CPL (bobot 3)
- K1.3: Publikasi & Kaji Ulang CPL (bobot 3)

**K2 — 5 sub-grup:**
- K2.1: Kurikulum (bobot 4)
- K2.2: Dosen (bobot 2)
- K2.3: Mahasiswa & Suasana Akademik (bobot 4)
- K2.4: Fasilitas (bobot 1)
- K2.5: Tanggung Jawab Institusi (bobot 2)

**K3 — 2 sub-grup:**
- K3.1: Proses Asesmen CPL (bobot 2)
- K3.2: Jaminan Pemenuhan CPL Lulusan (bobot 2)

**K4 — 2 sub-grup:**
- K4.1: Evaluasi Berkala (bobot 2)
- K4.2: Pemeliharaan Dokumen Evaluasi (bobot 2)

### Level 2: Sub-Kriteria (Submission Point, 29 item)

**Ini adalah unit terkecil yang:**
- ✅ Memiliki template items
- ✅ Dosen isi dan submit
- ✅ Validator review dan approve
- ✅ Dihitung dalam skor laporan

Distribusi: K1=8, K2=13, K3=4, K4=4, **Total=29**

## 2. Panduan Pengisian Submission

### 2.1 Alur Submission (Dosen)

```
DRAFT
  ↓ (Isi template, preview skor)
SUBMITTED
  ↓ (Tunggu review validator)
┌─ DITERIMA ✅ (Masuk laporan final)
│
├─ REVISI 🔄 (Kembali ke DRAFT, edit ulang, resubmit)
│
└─ DITOLAK ❌ (Perlu isi ulang dari awal)
```

### 2.2 Cara Mengisi Template

Setiap sub-kriteria Level 2 memiliki **3 tipe item template:**

#### Tipe 1: NARASI (bobot = 0, tidak hitung skor)
- **Fungsi:** Penjelasan kontekstual
- **Cara isi:** Ketik langsung di form textarea
- **Contoh:** "Deskripsi Profil Profesional Mandiri"
- ⚠️ **Wajib diisi tapi TIDAK pengaruh skor**

#### Tipe 2: UPLOAD (bobot = 30/60/70%, hitung skor)
- **Fungsi:** Bukti dokumen
- **Cara isi:** Upload file PDF/DOCX/XLSX
- **Aturan:**
  - Ukuran maksimal: 10 MB
  - Format: PDF, DOCX, XLSX, DOC
  - Validasi: File harus ada, tidak boleh kosong
- ✅ **Jika upload berhasil → Item dianggap TERISI (berkontribusi ke skor)**
- ❌ **Jika tidak upload → Item tidak terisi (skor 0)**

#### Tipe 3: CHECKLIST (bobot = 30%, hitung skor)
- **Fungsi:** Konfirmasi/pernyataan
- **Cara isi:** Centang checkbox ☑️
- **Contoh:** "PPM tersedia di website prodi"
- ✅ **Jika dicentang → Skor +30% (sesuai bobot)**
- ❌ **Jika tidak dicentang → Skor 0**

### 2.3 Pola Template Standar

Setiap sub-kriteria Level 2 mengikuti **4 pola template:**

#### Pola A: Standar (narasi + upload + checklist)
```
Item 1: NARASI         bobot = 0%   (Penjelasan)
Item 2: UPLOAD         bobot = 70%  (Dokumen utama)
Item 3: CHECKLIST      bobot = 30%  (Konfirmasi)
────────────────────────────────
TOTAL BOBOT WAJIB = 100
```

**Contoh K1.1.1 - PPM Prodi:**
- Item 1: Deskripsi PPM (narasi)
- Item 2: Dokumen Kurikulum/Panduan Akademik (upload, 70%)
- Item 3: Dokumen Renstra (upload, 30%)

#### Pola B: Dua Dokumen (narasi + upload + upload)
```
Item 1: NARASI         bobot = 0%   (Penjelasan)
Item 2: UPLOAD         bobot = 60%  (Dokumen utama)
Item 3: UPLOAD         bobot = 40%  (Dokumen pendukung)
────────────────────────────────
TOTAL BOBOT WAJIB = 100
```

**Contoh K1.2.2 - CPL:**
- Item 1: Rumusan CPL (narasi)
- Item 2: Dokumen CPL (upload, 60%)
- Item 3: Contoh RPS dengan CPL (upload, 40%)

#### Pola C: Kuantitatif (narasi + numerik + upload)
```
Item 1: NARASI         bobot = 0%   (Penjelasan)
Item 2: NUMERIK        bobot = 40%  (Angka/data)
Item 3: UPLOAD         bobot = 60%  (Bukti dokumen)
────────────────────────────────
TOTAL BOBOT WAJIB = 100
```

**Contoh K2.2.1 - Komposisi Dosen:**
- Item 1: Penjelasan dosen (narasi)
- Item 2: Jumlah dosen (numerik, ≥ nilai_min, 40%)
- Item 3: Dokumen data dosen (upload, 60%)

#### Pola D: Dokumen Saja (narasi + upload)
```
Item 1: NARASI         bobot = 0%   (Penjelasan)
Item 2: UPLOAD         bobot = 100% (Dokumen lengkap)
────────────────────────────────
TOTAL BOBOT WAJIB = 100
```

**Contoh K2.4.1 - Fasilitas:**
- Item 1: Penjelasan fasilitas (narasi)
- Item 2: Dokumen kepemilikan/SOP fasilitas (upload, 100%)

### 2.4 Threshold Skor Submission

**Sebelum Submit:**
- ✅ Skor sementara minimal **50%** untuk bisa submit
- ❌ Skor < 50% tidak boleh submit

**Setelah Submit:**
- Tunggu review validator
- Validator bisa: approve (diterima), revisi, atau tolak

## 4. Peran Validator

### 4.1 Dashboard Validator

Validator melihat:
- ✅ Daftar submission dengan status `submitted` (menunggu review)
- ✅ Riwayat review yang telah dilakukan
- ✅ Filter: by status, by prodi, by tanggal

### 4.2 Review Submission

Validator dapat:

#### 1️⃣ **APPROVE (Status → DITERIMA)**
- Klik tombol "Terima"
- ✅ Submission masuk ke laporan final
- ✅ Skor langsung dihitung di laporan prodi
- ℹ️ Komentar: opsional (untuk feedback positif)

#### 2️⃣ **REVISI (Status → REVISI)**
- Klik tombol "Revisi"
- 📝 **WAJIB isi komentar** (berisi instruksi perbaikan)
- Sistem auto-trigger: status → DRAFT (kembali ke dosen)
- Dosen dapat edit ulang & resubmit
- ℹ️ Komentar masuk notifikasi dosen

#### 3️⃣ **TOLAK (Status → DITOLAK)**
- Klik tombol "Tolak"
- 📝 **WAJIB isi komentar** (alasan penolakan)
- ❌ Submission tidak masuk laporan
- ℹ️ Dosen harus buat submission baru dari awal
- Submission lama HILANG dari perhitungan skor

### 4.3 Checklist Validator Saat Review

**Sebelum approve, validator harus cek:**

✅ **Narasi (Item 1)**
- Apakah penjelasan cukup detail & sesuai konteks?
- Apakah menjawab pertanyaan di keterangan?

✅ **Upload (Item 2, 3)**
- Apakah dokumen ter-upload?
- Apakah file format benar (PDF/DOCX)?
- Apakah isi dokumen sesuai dengan label?
- Apakah dokumen asli/resmi (bukan draft)?

✅ **Checklist (jika ada Item 3)**
- Apakah checked box mencerminkan kenyataan?
- Apakah ada bukti pendukung?

### 4.4 Status Flow Setelah Review

```
SUBMITTED (menunggu validator)
       ↓
    ┌──┴──────────┬──────────────┐
    ↓             ↓              ↓
 DITERIMA      REVISI        DITOLAK
  (final)      (→ DRAFT)     (discard)
    ↓             ↓              ↓
 Masuk       Dosen edit    Mulai ulang
 laporan      & resubmit    dari awal
```

## 5. Rumus Perhitungan Skor

### 5.1 Formula Dasar — Skor per Submission (Level 2)

```
Skor_submission (%) = SUM(bobot_item × status_terisi_item) / SUM(bobot_item_total)

Dimana:
  bobot_item = bobot masing-masing item (0%, 30%, 40%, 50%, 60%, 70%, 100%)
  status_terisi = 1 jika item terisi, 0 jika kosong
  SUM(bobot_item_total) = total bobot WAJIB per sub-kriteria = 100%
```

### 5.2 Status Terisi per Tipe Item

| Tipe Item | Kondisi Terisi | Kontribusi Skor |
|-----------|---|---|
| **NARASI** | `nilai_teks tidak kosong` | ❌ TIDAK (bobot=0) |
| **UPLOAD** | File minimal 1 terupload | ✅ YA (sesuai bobot) |
| **NUMERIK** | `nilai ≥ nilai_min` | ✅ YA (sesuai bobot) |
| **CHECKLIST** | Box di-check ✅ | ✅ YA (sesuai bobot) |

### 5.3 Aggregation: Level 2 → Level 1 → Level 0

**Level 2 (Individual Sub-Kriteria):** Skor 0-100%
```
Skor_K1.1.1 = (70 + 30) / 100 = 100%  ← Upload 2 dokumen berhasil
```

**Level 1 (Pengelompok):** Weighted Average dari Level 2
```
Skor_K1.1 = (Skor_K1.1.1 × 1 + Skor_K1.1.2 × 1) / (1 + 1)
         = (100% × 1 + 80% × 1) / 2
         = 180 / 2 = 90%

Bobot Level 2 = 1 (sama untuk semua 29 item)
```

**Level 0 (Kriteria Utama):** Weighted Average dari Level 1
```
Skor_K1 = (Skor_K1.1 × 2 + Skor_K1.2 × 3 + Skor_K1.3 × 3) / (2 + 3 + 3)
        = (90% × 2 + 95% × 3 + 85% × 3) / 8
        = (180 + 285 + 255) / 8
        = 720 / 8 = 90%

Bobot Level 1 = jumlah Level 2 di dalamnya
```

**Grand Total Prodi:** Weighted Average Level 0
```
Skor_Total_TRIN = (Skor_K1 × 28 + Skor_K2 × 45 + Skor_K3 × 14 + Skor_K4 × 13) / 100

Bobot Level 0 = proporsional jumlah Level 2 di dalamnya
```

### 5.4 Threshold & Status Kesiapan

| Skor | Status | Label UI |
|------|--------|----------|
| ≥ 80% | **Sangat Siap** | ✅ FULLY COMPLIANT |
| 75–79.99% | **Siap** | ✅ COMPLIANT |
| 60–74.99% | **Hampir Siap** | ⚠️ ALMOST COMPLIANT |
| < 60% | **Perlu Perbaikan** | ❌ NOT COMPLIANT |

## 6. Contoh Kalkulasi Skor Lengkap

### Scenario: Prodi TRIN Submission 5 item K1

Misalkan TRIN sudah submit 5 sub-kriteria dari K1 (sisanya masih draft):

#### **Step 1: Hitung Skor Per Submission (Level 2)**

**K1.1.1 — PPM Prodi:**
```
Item 1 (narasi 0%):  Diisi tapi bobot=0        → Kontribusi: 0
Item 2 (upload 70%): Dokumen upload ✅         → Kontribusi: 70
Item 3 (upload 30%): Tidak upload ❌           → Kontribusi: 0
───────────────────────────────────────────────────
Skor_K1.1.1 = (70 + 0) / 100 = 70%
```

**K1.1.2 — Proses Penyusunan PPM:**
```
Item 1 (narasi 0%):  Diisi tapi bobot=0        → Kontribusi: 0
Item 2 (upload 50%): Upload ✅                 → Kontribusi: 50
Item 3 (upload 50%): Upload ✅                 → Kontribusi: 50
───────────────────────────────────────────────────
Skor_K1.1.2 = (50 + 50) / 100 = 100%
```

**K1.2.1 — Penyebaran PPM (Pola A):**
```
Item 1 (narasi 0%):     Diisi          → Kontribusi: 0
Item 2 (upload 70%):    Upload ✅      → Kontribusi: 70
Item 3 (checklist 30%): Checked ✅     → Kontribusi: 30
───────────────────────────────────────────────────
Skor_K1.2.1 = (70 + 30) / 100 = 100%
```

**K1.2.2 — CPL (Pola B):**
```
Item 1 (narasi 0%):   Diisi          → Kontribusi: 0
Item 2 (upload 60%):  Upload ✅      → Kontribusi: 60
Item 3 (upload 40%):  Tidak upload ❌ → Kontribusi: 0
───────────────────────────────────────────────────
Skor_K1.2.2 = (60 + 0) / 100 = 60%
```

**K1.2.3 — CPL Tambahan (Pola A):**
```
Item 1 (narasi 0%):     Diisi          → Kontribusi: 0
Item 2 (upload 70%):    Upload ✅      → Kontribusi: 70
Item 3 (checklist 30%): Tidak check ❌  → Kontribusi: 0
───────────────────────────────────────────────────
Skor_K1.2.3 = (70 + 0) / 100 = 70%
```

#### **Step 2: Aggregasi Level 2 → Level 1 (Weighted Average)**

**K1.1 (Penetapan PPM):**
```
Skor_K1.1 = (Skor_K1.1.1 × 1 + Skor_K1.1.2 × 1) / (1 + 1)
         = (70% × 1 + 100% × 1) / 2
         = 170 / 2
         = 85%
```

**K1.2 (Penyebaran PPM & CPL):**
```
Skor_K1.2 = (Skor_K1.2.1 × 1 + Skor_K1.2.2 × 1 + Skor_K1.2.3 × 1) / (1 + 1 + 1)
         = (100% × 1 + 60% × 1 + 70% × 1) / 3
         = 230 / 3
         = 76.67%
```

**K1.3 (Publikasi & Kaji Ulang CPL):**
```
Status: Belum ada submission (DRAFT) → Skor = 0% atau diabaikan
```

#### **Step 3: Aggregasi Level 1 → Level 0 (Weighted Average)**

**K1 (Orientasi Kompetensi Lulusan):**
```
Hanya yang status DITERIMA dihitung:

Skor_K1 = (Skor_K1.1 × 2 + Skor_K1.2 × 3) / (2 + 3)
        = (85% × 2 + 76.67% × 3) / 5
        = (170 + 230) / 5
        = 400 / 5
        = 80%

Catatan: K1.3 belum ada status DITERIMA, jadi tidak dihitung
```

#### **Step 4: Asumsi K2, K3, K4 & Grand Total**

Misalkan status submission lain:

| Kriteria | Diterima? | Skor Asumsi |
|----------|-----------|------------|
| K1 | 5 of 8 | 80% |
| K2 | 0 of 13 | — (belum dihitung) |
| K3 | 0 of 4 | — (belum dihitung) |
| K4 | 0 of 4 | — (belum dihitung) |

**Grand Total (hanya K1 yg ada):**
```
Skor_Total_TRIN = (80% × 28) / 28 = 80%

Status: HAMPIR SIAP ⚠️ (karena K2, K3, K4 belum submission)
```

**Jika K2, K3, K4 semua submit & approve (asumsi 85%, 90%, 75%):**
```
Skor_Total = (80% × 28 + 85% × 45 + 90% × 14 + 75% × 13) / 100
          = (2240 + 3825 + 1260 + 975) / 100
          = 8300 / 100
          = 83%

Status: SANGAT SIAP ✅
```
